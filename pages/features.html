Features

  • Coding of video and audio files
    using an integrated playback at various speeds (from x0.1 to x30).
  • Event logging for live observations
    with the desktop program or the BORIS App for Android
  • Up to 8 media files can be synchronously played.
    A time offset allows to synchronize all players.
  • Frame-by-frame mode (forward and backward).
  • Unlimited number of behaviors.
    Behaviors can be defined as State events (with duration) or Point events (no duration) and can be organized in behavioral categories.
  • Unlimited number of modifiers for each behavior.
    The modifiers can be of 3 types (single selection, multiple selection and numeric) and can be organized in an unlimited number of sets.
  • Unlimited number of subjects.
    The focal subject selection can be made from a Subjects pad (for touch screens).
  • Unlimited number of independent variables
    Variables can be of various types: text, numeric, timestamp or set of values
  • Focus video area (with a single click on video)
  • Video transcoding functions: Re-size / re-encode, rotate
  • The behavior coding can be done with keyboard, mouse, by using a behaviors coding map or a coding pad (for touch screens)
  • External data visualization from plain text files (TSV, CSV ...).
    Native or custom converters can be used for converting external data timestamp in seconds.
  • Availability of an Android app for live observations (BORIS App)
  • Sound spectrogram visualization
  • Geometric measurements
    in frame-by-frame mode (distance, area and angle).
  • Coefficient of inter-rater reliability (IRR):
    • Cohen's Kappa.
  • Transition matrix
  • Flow diagram
    The GraphViz program is required.
  • More media can be enqueue for playing
    the recorded time will be the cumulative time over all media.
  • Coded events can be exported in various formats for further analysis:
    • Plain text formats (TSV, CSV and HTML)
    • Spreadsheets (Open Document ODS, Microsoft-Excel XLSX and XLS)
    • Structured Query Language (SQL)
    • SDIS for analysis with the GSEQ program.
    • Praat TextGrid
    • Behavioral strings for analysis with Behatrix or other tools.
    • JWatcher
  • Time budget analysis
    • 3 time budgets are available:
      • Time budget
      • Time budget by behavioral category
      • Synthetic time budget.
    • Results can be exported in various formats:
      • Plain text formats (TSV, CSV, HTML)
      • Spreadsheets (Open Document, Microsoft-Excel XLSX and XLS)
  • Graphical representation of coded data
    The plots can be exported in various formats to be included in scientific presentations:
    • SVG format
    • Portable Document Format (PDF)
    • Postscript (PS or EPS)
    • Portable Network Graphics (PNG)
    • JPEG
    • Tagged Image File Format (TIFF)
  • Media snapshot
  • Creation of video subtitles based on coded events and subjects
  • Use of open format for storing data (JSON) to allow further analysis
  • Import of Global Definition File from JWatcher
  • Import of OTX/OTB template file from Noldus The Observer XT
